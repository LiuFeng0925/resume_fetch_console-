邮箱简历附件自动抓取 · mockup

从多个企业邮箱 INBOX 定时/手动拉取应聘邮件，按标题正则识别后，将 PDF 等附件保存到各账号独立目录。 本文档说明整体架构、多账号配置、运行方式与交付形态——不是产品 UI 设计稿。

§0 最终交付形态

最重要
结论：Python 命令行脚本 + cron，不是网站/App/安装包。
一次 python main.py run 按顺序处理 accounts 里配置的所有邮箱；cron 每小时调同一命令。成果在各账号的本地文件夹里。
✅ v1 交付
CLI + cron + 多账号 YAML

手动 run 或 cron 定时；多邮箱写在 accounts；SQLite 统一去重。

不是
Web UI / 桌面 App

无浏览器界面、无 .app / dmg。改 config.yaml，看文件夹和日志。

§2 已确认的关键决策

决策点选项备注
邮箱数量多账号（accounts 列表）一份 config，一次 run 顺序处理；v1 不并行
连接方式IMAP over SSL每账号独立 password_env
扫描范围仅 INBOX每账号可写 mailbox: INBOX
识别规则邮件标题正则（全局共用）subject_patterns 对所有账号生效
附件筛选扩展名白名单（全局共用）默认 .pdf
保存目录每账号独立 download.path如 ~/Resumes/hr 与 ~/Resumes/recruiter2
文件命名{YYYYMMDD}_{原附件名}各账号目录内平铺存储
去重(account_name, Message-ID)SQLite 联合主键；账号间互不影响
watermark每账号独立首次运行不回溯该账号历史邮件
定时每小时 cron0 * * * *
失败策略单账号失败继续其余任一失败 → 进程 exit 非零

§3 多邮箱方案 A

accounts 列表
config.yaml
├── accounts[]                    ← 用户主要编辑区
│   ├── [0] name: hr-boss
│   │       imap + password_env   ← IMAP_PASSWORD_HR
│   │       download.path         ← ~/Resumes/hr
│   └── [1] name: recruiter-2
│           imap + password_env   ← IMAP_PASSWORD_R2
│           download.path         ← ~/Resumes/recruiter2
├── subject_patterns[]            ← 全局，所有账号共用
├── attachment_extensions[]       ← 全局
├── state.db_path                 ← 一个 SQLite，表内按 account_name 区分
└── log.path

账号与目录对应关系

name邮箱密码环境变量简历目录

§4 上下游全景图

  cron / 手动 run
        │
        ▼
  ┌─────────────────────────────────────────────────────────────────┐
  │  main.py run  ──▶  for account in accounts:  （顺序，不并行）       │
  │                    │                                             │
  │                    ├─▶ IMAP 连接该账号 INBOX                       │
  │                    ├─▶ 跳过 (account, Message-ID) 已在 SQLite    │
  │                    ├─▶ 标题正则匹配 → 下载到 account.download.path │
  │                    ├─▶ 写入 SQLite + 更新该账号 watermark         │
  │                    └─▶ 断开 IMAP → 下一账号                      │
  └─────────────────────────────────────────────────────────────────┘
        │                              │
        ▼                              ▼
  企业邮箱 × N                    本地目录 × N + processed.db + logs
关键不变式： (1) 只读邮箱，不改已读状态； (2) 去重键 (account_name, message_id)； (3) 每账号 watermark 独立，首次不回溯； (4) 单账号失败不阻断其他账号。

§5 单次 run 流程

1
加载配置

解析 accounts[]、全局正则；校验 name 唯一、env 存在

2
遍历账号

按列表顺序；--account 时只跑一个

3
IMAP 拉信

该账号 INBOX，watermark 之后的新邮件

4
匹配 & 下载

标题正则 + 扩展名白名单 → 该账号 download.path

5
记录去重

SQLite 写入 (account_name, message_id)

6
汇总退出

每账号摘要 + 总摘要；有失败则 exit ≠ 0

分支逻辑

条件动作
(account, Message-ID) 已存在跳过该邮件
标题未命中正则跳过（v1 仍标记已处理，避免重复扫描）
命中但无白名单附件标记已处理
本地同名文件追加 _2、_3
当前账号 IMAP 失败记日志，继续下一账号
任一账号失败全部跑完后 exit code 非零

§7 配置契约 (config.yaml)

accounts:
  - name: hr-boss
    imap:
      host: imap.exmail.qq.com
      port: 993
      ssl: true
      username: hr@company.com
      password_env: IMAP_PASSWORD_HR      # export IMAP_PASSWORD_HR='...'
    mailbox: INBOX
    download:
      path: /Users/liufeng/Resumes/hr

  - name: recruiter-2
    imap:
      host: imap.exmail.qq.com
      port: 993
      ssl: true
      username: recruiter2@company.com
      password_env: IMAP_PASSWORD_R2
    mailbox: INBOX
    download:
      path: /Users/liufeng/Resumes/recruiter2

subject_patterns:                         # 全局，所有账号共用
  - ".*应聘.*【BOSS直聘】"
  - ".*应聘.*"

attachment_extensions:
  - ".pdf"

state:
  db_path: ./data/processed.db

log:
  path: ./logs/resume-fetch.log

完整模板见项目内 config.yaml.example。用户复制为 config.yaml 后主要改 accounts 里各账号的 IMAP 与 download.path。

§9 部署三步走

Step 1
配置

cp config.yaml.example config.yaml

Step 2
手动验证

注入各账号 password_env
python main.py run

Step 3
注册 cron

0 * * * * cd /path/to/邮箱抓取简历 && IMAP_PASSWORD_HR='***' IMAP_PASSWORD_R2='***' python3 main.py run >> logs/cron.log 2>&1

§10 模块划分

模块职责多账号相关
Config Loader解析 accounts[]，校验 name 唯一、必填项返回账号列表 + 全局规则
Processed Mail StoreSQLite 去重、watermark键含 account_name
IMAP Mail Fetcher每账号实例化，拉 INBOX一账号一连接，顺序执行
Subject Matcher / Attachment Filter全局规则所有账号共用
Attachment Downloader写入该账号 download.path路径来自当前 account
Job Orchestratorrun [--account NAME]外层 for 循环 + 失败继续
